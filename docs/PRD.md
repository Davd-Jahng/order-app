# COZY 커피 주문 앱 — PRD (Product Requirements Document)

---

## Part I. 개요

### 1.1 문서 정보

| 항목 | 내용 |
|------|------|
| 프로젝트명 | COZY — 커피 주문 앱 |
| 문서 목적 | 주문하기·관리자 화면 기능 및 데이터 요구사항 정의 |
| 범위 | 풀스택 웹앱 (주문 플로우 + 관리자 운영) |

### 1.2 프로젝트 목적

- **사용자**: 커피 메뉴 선택, 옵션 선택, 장바구니 담기, 주문 제출
- **관리자**: 주문 상태 관리, 재고 조회·조정, 대시보드 요약 확인

### 1.3 개발 범위

| 구분 | 내용 |
|------|------|
| **포함** | 주문하기 화면, 관리자 화면, 메뉴/옵션/주문/재고 CRUD, 공통 네비게이션 |
| **제외** | 사용자 인증, 결제 기능 |
| **메뉴** | 커피 메뉴만 대상 (추후 확장 가능) |

### 1.4 가이드라인 및 제약사항

- 프론트엔드와 백엔드는 **별도 디렉터리/프로젝트**로 개발
- 기본 웹 기술 위주로 구현

## Part II. 공통 사양

### 2.1 공통 UI — 상단 헤더·네비게이션

- **브랜드**: 좌측에 "COZY" 표시
- **탭**

  - **관리자**: 관리자 화면으로 이동 (해당 화면에서 선택/강조)
- 두 화면 모두 동일한 헤더 레이아웃 사용

## Part III. 기능 요구사항

### 3. 주문하기 화면

#### 3.1 화면 구성

| 영역 | 설명 |
|------|------|
| 상단 헤더 | [2.1] 공통 헤더 (주문하기 탭 강조) |
| 메뉴 영역 | 상품 카드 가로 배치 |
| 장바구니 영역 | 담긴 항목 목록, 총 금액, 주문하기 버튼 |

#### 3.2 메뉴(상품) 카드

- **구성 요소**: 상품 이미지(또는 플레이스홀더), 상품명, 기본 가격, 간단한 설명, 옵션(체크박스), 담기 버튼
- **옵션 예시**: 샷 추가 (+500원), 시럽 추가 (+0원) — 선택 시 단가에 반영
- **담기**: 클릭 시 선택 옵션을 반영한 항목이 장바구니에 추가

**비즈니스 규칙**

- 동일 상품 + 동일 옵션 조합 → 장바구니에서 **수량 합산**
- 단가 = 기본가 + 옵션 추가금

#### 3.3 장바구니

- **목록**: `상품명 (옵션명) X 수량` + 항목별 금액
- **총 금액**: 합계 표시
- **주문하기 버튼**: 클릭 시 장바구니 내용으로 주문 생성 (결제·인증 없음)

### 4. 관리자 화면

#### 4.1 화면 구성

| 영역 | 설명 |
|------|------|
| 상단 헤더 | [2.1] 공통 헤더 (관리자 탭 강조) |
| 관리자 대시보드 | 주문 상태별 건수 요약 |
| 재고 현황 | 상품별 재고 수량 및 증감 조절 |
| 주문 현황 | 개별 주문 목록 및 상태 변경 |

#### 4.2 관리자 대시보드

- **표시 형식**: `총 주문 N / 주문 접수 N / 제조 중 N / 제조 완료 N`
- **의미**: 주문 상태별 건수 집계 (총 주문, 접수, 제조 중, 제조 완료)

#### 4.3 재고 현황

- **카드**: 상품명, 현재 재고 수량(예: 10개), `-` / 수량 / `+` 버튼
- **동작**: `+` 1 증가, `-` 1 감소 (재고 0 미만 불가)
- **저장**: 즉시 반영 또는 저장 버튼 — 구현 시 결정

#### 4.4 주문 현황

- **행 정보**: 주문 시각, 주문 내용(상품 x 수량), 금액, 상태 버튼(예: 주문 접수)
- **상태 흐름**: 신규 → 주문 접수 → 제조 중 → 제조 완료
- **정렬**: 최신순 등 (구현 시 결정)

**비기능**

- 재고 수량 ≥ 0 유지

## Part IV. 데이터 요구사항

### 5.1 엔티티·속성 요약

| 엔티티 | 주요 속성 | 비고 |
|--------|-----------|------|
| **메뉴(상품)** | 이름, 기본 가격, 설명, 이미지 URL(또는 경로) | |
| **옵션** | 옵션명, 추가 금액 (0원 가능) | 상품과 연관 |
| **장바구니** | 상품 ID, 옵션(또는 옵션 조합), 수량, 계산된 단가 | 클라이언트/세션 또는 DB |
| **주문** | 주문 시각, 주문 항목 스냅샷(상품/옵션/수량/금액), 총 금액, **상태** | 상태: 접수대기/접수/제조중/제조완료 |
| **재고** | 상품 ID, 현재 수량 | 상품별 1행 |

### 5.2 주문 상태

- 접수 대기 → 주문 접수 → 제조 중 → 제조 완료  
- 대시보드는 위 상태별 건수 조회·집계

## Part VI. 부록

### 7. 용어 정의

| 용어 | 설명 |
|------|------|
| 담기 | 메뉴+옵션을 장바구니에 추가하는 행위 |
| 주문하기(버튼) | 장바구니 내용을 한 건의 주문으로 제출하는 행위 |
| 옵션 | 샷 추가·시럽 추가 등 상품별 추가 선택 및 추가 금액 |
| 주문 접수 | 관리자가 주문을 확인하고 접수 상태로 변경하는 행위 |
| 제조 중 | 접수 후 음료 제조가 진행 중인 상태 |
| 제조 완료 | 주문 음료 제조가 완료된 상태 |
| 재고 현황 | 메뉴(상품)별 보유 수량 및 증감 조정 기능 |
